Pipeline Description

This application is part of a DevOps pipeline designed to automate the processes of Continuous Integration (CI) and Continuous Delivery (CD) for a microservices-based web application. The pipeline is configured as follows:

  • Source Control: The application code is stored in a GitHub repository.
  • Continuous Integration: Jenkins is used to automate the CI process, which includes cloning the repository and building a Docker image.
  • Automated Testing: Automated tests are integrated into the pipeline to ensure code quality and functionality.
  • Continuous Delivery: The Docker image is pushed to Docker Hub, and the application is deployed to a local Kubernetes cluster managed by Minikube.
  • Deployment: Kubernetes (minikube) deployment and service configurations are used to manage the application instances and expose them to the network.

With this pipeline, we achieve a streamlined and automated workflow that ensures consistent application delivery and deployment.
